Designer's critique — read before you fall in love with it

1. The seam puts power-mods in the wrong place. A haste/power booster wants to live inside one hero's board. Parked in the seam it's forced to straddle two heroes — geometrically confused. Only combine-type modifiers truly belong on a seam.
2. Hero 2 is overpowered; Hero 3 is starved. H2's centre tile touches all 4 modifiers; H3 has no seam below it, so it can only ever be bridged into by H2. The board isn't symmetric — three heroes are not equal citizens.
3. Only 4 bridge points for 9 elements. Most tiles just fire raw; the interesting combos are rare. The "craft puzzle" is thinner than it looks.
4. Fixed connector-shapes + 4 slots ≈ solve-once. Without rotation or churn there are few meaningful arrangements. Depth needs either rotatable modifiers or per-wave matchup pressure. (Full critique in chat.)
Decide before you lock

Three slot models — A / B / C, side by side

Same elements + the same 6 connector-logics. What changes is where modifiers are allowed to live — and that single choice decides whether your tug-of-war (cluster for buffs vs. insert a modifier) exists at all.

Your tug-of-war question, answered up front

The tension you want — keep same elements adjacent for a cluster buff, OR spend that space on a modifier that transforms them — only exists if elements and modifiers compete for the same cells. In A and B they don't (dedicated rows vs. dedicated seams), so there's no tug-of-war — you just fill each slot type. Only C (unified grid) creates it. So this question and the A/B/C choice are the same decision: dedicated slots (legible, no tension) vs. unified grid (tense, deeper, busier).
A

Combiner-only seams

Board modifiers are combiners only (Conduit/Splitter/Prism) and live in the seams. Power/speed is off the board (hero leveling). Cleanest, most casual — keeps "a modifier = it combines what it touches." Cluster buff = same element adjacent in a row.

B

Combiner seams + a per-hero power slot

Each hero gets its own power slot (buffs that whole hero — the GD speed-gear, unambiguously owned). Seams stay combiner-only. Both classes get a coherent home; costs one extra slot + a second slot type to teach.

C

Unified grid — the tug-of-war

One hex board, any cell = element OR modifier. Cluster same elements for buffs, but every modifier you drop costs a cell and can break a cluster. Use the two preset buttons to feel it: Cluster (3 buffs, 0 combos) vs. Bridge (0 buffs, 1 combo). This is the deepest and the busiest.

Path Y — modifier converts its cluster · 5-cell scarce board · sim-verified

Cells are scarce. Cluster same elements to fire as one tick · merge 3 alike (tap the glow) for the damage upgrade + free cells · place a directional Key so its arrows point at two different elements and it fuses them into a combo that bypasses resists (the epic Star fuses three). The mana bar grows one segment per tick — one per cluster / combo / lone tile — and fills across them before the hero attacks, so fewer groups = a shorter bar = faster attacks (a cluster or combo = one segment). Press ⚔ Battle — combat is automatic.

Model (Path Y): a Key doesn't fire its own shot — it converts the element clusters its arrows point at into combo-type damage (scales with cluster size, bypasses resist, ×3 vs a mega-weak enemy). Keys are directional: a 2-arrow Key fuses 2 tiles, the epic Star fuses 3 — a tile no arrow points at is left raw, so there is no hidden priority. A cluster is keyed once. The Splitter instead doubles its pointed tiles as raw damage (no combo, no resist-bypass). Mana = ticks: the bar gains one segment per cluster, combo, or lone tile — it fills across all the segments, then the hero attacks. A cluster or combo counts as ONE segment no matter how many tiles, so fewer / bigger groups = a shorter bar = faster attacks (a combo turns 2 segments into 1). Combat runs only after you press ⚔ Battle. Numbers placeholder.
