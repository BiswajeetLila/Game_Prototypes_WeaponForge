Executive summary

Between roughly May and mid-June 2026 the WeaponCraft idea cycled through six numbered variants across two engines (HTML/JS prototypes → Godot 4.6 Mono), one explicit post-mortem, two live forks, one pivot, and one autonomous overnight build. Today, only one variant is forward-active (6_WeaponForge_TFTransistor); two are frozen reference builds (2_Weaponcraft_Godot, 5_WeaponForge_Honkai_Godot); one is archived as a learning artifact (4_WeaponCraft_ForgeLoop); and two early variants survive as design-history reference.

The throughline: we kept trying to find the moment that makes a craft feel like a craft — and we kept discovering that the answer was upstream of whatever we were currently polishing. Anatomical part-slot crafting (Head + Hilt + Rune) felt mechanical; modal minigames felt like four games glued together; weapon-gacha-on-locked-roster felt mathematically clean but lacked the "do something with your hands" beat; the current Transistor-style Function Matrix is the closest yet but is still being grilled on its power-gating layer.

Three durable lessons sit underneath all of it. First, sensory polish is not an engagement fix: variants 2 and 4 both proved that a tight per-tap feedback layer can ship green and still bounce playtesters when the underlying loop is wrong. Second, tests that assert data shape do not test the game — they assert that the rows in your config file are spelled correctly while the combat code never reads them. Third, discipline beats velocity: the variant that survives (variant 6) is the one with TDD + RED→GREEN + adversarial-review workflows enforced; the variants that died with the most code shipped per day did not have those guardrails.

For the new prototyper

If you read one section after this summary, read §5 "What failed" and then §6 "Engineering gotchas". The wins are reproducible from the docs; the losses are not, and the gotchas will eat a day of your life if you have to rediscover them.

Variant-by-variant retrospective

1 — Robotek_WeaponCraft (canonical base)

The bet. A casual-mobile fantasy auto-battler in the Wittle Defender / AFK Journey / Hero Wars audience pocket. Take Robotek's turn-based send-my-unit-out cadence and bolt a TFT-style shop of weapon parts onto it. Move the "randomized dice" out of combat into craft. Three heroes, side-view single lane, boss every 5–6 waves, run ≈ 3–4 minutes.

What stuck. The skeleton from this GDD never died. Every later variant inherited: 3-hero party, ult-on-tap, side-view single-lane combat (until variant 3 / 6 broke that), TFT-shop parts feed, boss-affinity counter-build, persistent weapons across waves, ~3–4 min stages.

What was structurally wrong (in hindsight, only visible from variant 6's pivot addendum). The Head + Hilt + Rune anatomy means a player needs specific part types to assemble a weapon — and that puts them at the mercy of shop RNG. Worse, side-view single-lane combat gives the player no real-time agency, so the crafting layer has to carry 100% of the cognitive load — and an anatomical crafting layer is too restrictive to do that. We did not see this problem in variant 1 because variant 1 was a GDD, not a build.

2a — WeaponCraft_Base (HTML/JS)

The bet. Ship variant 1 fast in HTML/JS. Get it in front of a playtester. Don't over-engineer.

What worked. First real playable. The parchment-brass theme, hero card, ult gauge, affinity telegraph, console event schema (window.__forgeLoopStats()-style snapshot in the later sibling), and the BASE-A1 .html as a single self-contained fork target all became downstream-reusable. The reveal climax (the 4-second anvil-RING that variant 4 borrowed) was first prototyped here.

What didn't. Menu-based crafting got boring after ~2 minutes. The playtester signal was unambiguous; the diagnosis was less so. We initially read it as a juice problem (→ variant 4) when it was a structural agency problem (→ variant 6 finally got it).

2b — Weaponcraft_Godot (port + meta layer)

The bet. Port BASE-A1 to Godot 4.6 Mono so the project has a real engine substrate. Add a persistent hero-squad gacha meta layer (HOME → squad-select → run → result → scripted pull). Get to P0.

What worked. This is the variant where the engineering discipline crystallized. TDD per task, RED→GREEN per checkpoint, commit + push per green task, headless test sweeps (exit code = failure count). The autonomous overnight run on the night of 11→12 Jun produced 210/210 tests across 7 suites and a playable build — and crucially, captured the running judgment calls into a NIGHT-LOG.md rather than burning them on resume. Save-schema versioning (v1 → v2 with explicit migration test) was established here. The "art = layout/palette truth" decision (don't pixel-cut UI chrome from AI screens; build from existing portraits + theme matching) is good guidance.

What didn't. The structural problem from 2a was inherited unchanged. Polish on the meta layer (rich victory screen, scripted Vex pull, scout intel strip, level-up rows) raised the floor of how the game looked without raising the ceiling of how it played. Within hours of shipping P0 we were writing the pivot addendum.

Smell

If you find yourself polishing the meta layer ahead of a feel-gate playtest of the core loop, the meta-layer work is probably procrastination on a core-loop problem you already know exists. Variant 2b shipped P0 and the pivot was decided the same day.

3 — WeaponCraft_RealTime (design-only)

The bet. What if the combat layer became continuous real-time (Wittle / Brotato cadence) and the shop became a scrolling carousel (5 tiles, oldest expires every 5s)? Real-time would give the player attention bandwidth back; the carousel would force "watch what's coming" decisions.

Why it died at spec. The variant never got a build. Looking back: the GDD is thoughtful — pulse-rhythm difficulty (15s quiet / 10s spike), revive-on-potion-click, 0:00→Bran 1:00→Elara 2:00→Vex hero unlock ramp. But it doesn't solve the structural anatomical-crafting problem from variant 2, it just changes the combat container. The Wittle-inversion fork (variant 5) took a more disruptive bet (locked roster + weapon gacha + Catalyst element pairs) and that crowded variant 3 out.

Survivors. The carousel-shop idea is still latent in variant 6's "slow-populate 2/3/2" shop drip. The pulse-rhythm difficulty curve is the kind of thing variant 6 will need when it starts tuning feel.

4 — WeaponCraft_ForgeLoop (the explicit failure)

The most important variant in this retrospective is the one that died. ForgeLoop has the only formal POST_MORTEM.md in the repo. The five-point breakdown is verbatim-worth-quoting:

"We don't want minigames inside a game. It's too much effort, and people churn out fast. We need things to flow cohesively in one game."Playtester verdict, ForgeLoop POST_MORTEM.md (25 May 2026)

The bet. Menu-based crafting from variants 2a/2b got boring → minigames-as-juice will re-engage players. Build a Loop-Hero-style 8-cell hex ring; 6 customizable cells; player drags station tiles (Bellows / Anvil / Quench / Engraver); a glowing ingot laps the ring; each station triggers a 3–7 s modal minigame; lap completes → 4-second anvil-RING reveal climax → weapon snaps onto hero. Forge Again? (5🪙) re-enters the loop and stacks stats.

What shipped. 14 commits, ~1900 lines in a single self-contained HTML, four working minigames, ring geometry + drag-drop placement, ingot travel animation, the reveal climax, hero card weapon mount with per-strike pulse and ★ crit markers. The Anvil minigame got a 5-layer feedback pass (banner / zone flash / sparks / hammer pulse / shake on miss).

Why it failed (the five-point breakdown).

  1. Minigames are micro-games, not gameplay. Four chained minigames per craft = 20+ s of being yanked into mode-switched sub-experiences. Players described it as playing four games in series.
  2. Modal pause kills momentum. A craft should feel like one continuous gesture; ForgeLoop's craft was a sequence of disconnected discrete tasks.
  3. Mastery cost too high for too little payoff. Each minigame had its own muscle memory; the interesting decision (which station, what order) sat behind a layer of execution skill that didn't reward growth — once you can hit 5/5 Anvil, it becomes a tax.
  4. Churn risk. ~25–35 s first forge + 4-s reveal + combat. Three of those seconds are spent on a modal minigame before the player has seen combat happen. Mobile players have ~10–15 s to decide if a session is worth continuing.
  5. Polish wasn't the bottleneck. The 5-layer Anvil feedback pass felt sharp and still bounced testers. "Polish-is-not-fun-fix" was discovered the hard way here.

What lives on. The post-mortem explicitly carries forward: the 4-second reveal climax pattern (every variant should have a payoff moment of similar shape), the weapon-name generator (Crude/Honed/Tempered/Masterwork × Element × Base), the SVG sprite composite (layered blade + tint + glyph), the spatial-combo seed (Bellows-before-Anvil = +30% Hot Strike was the most interesting decision the build offered), and the principle of depth-per-weapon over breadth-of-weapons.

5 — WeaponForge_Honkai_Godot (the Wittle-inversion fork)

The bet. Pull weapons, not heroes. Lock the roster at 7 (Bran / Elara / Vex + Hot Paladin via Stage-3 cinematic + 2nd Rogue + 2nd Mage + Hot Assassin; 3 deploy per stage). The moat is the combination of equipment-gacha (precedented; Archero $263M) with a story-locked roster (unprecedented). Add Catalyst element-pair compounds (renamed from Resonance because Habby owns that term via Archero 2). Stages = 5 waves with W5 boss (compressed from the design's 15-wave shape).

What shipped. A surprisingly complete prototype. By 10 Jun: Catalyst v1 with 10 element-pair compounds (Firestorm, Wildfire, Plasma, Blizzard, Glacial Storm, Stormfront, + 4 Earth-gated at S10); Ember pull currency separated from gems; Scripted Pacing Rework with the Hot Paladin Stage-3 scripted-defeat cinematic, Helios Cleaver scripted-grant, 4 new light Catalyst pairs (Solar Flare / Halo Bloom / Plasma Arc / Auroral Veil); a teammate pitch deck (HTML, Bran 5-tier portrait scrubber as the unforgettable hook); ~640 tests baseline; save schema reached v6 with explicit migrations at each bump.

What worked. The structural process discipline is the cleanest of any variant. The 5_/CLAUDE.md codifies it: brainstorm → spec → plan → TDD → execute. Save schema policy: every bump = new _migrate_vN_to_vN+1() + a TestAccountState case that loads an old-format dict and asserts the v+1 defaults. Numbers policy: numbers in code = starting values, tune via playtest. No magic numbers; ## Why N comments when a value deviates from the spec. Stage-1 combat is a frozen contract — any combat-math change must keep stage-1 green or explicitly rebaseline.

What didn't. The variant is the longest design-doc-to-build ratio of any variant; the prototype has not yet hit its exit gates (D1 ≥ 35% / FM-8 hero-bond probe ≥ 6/10 / ad CPI ≤ $2.80). The pre-flight gates (Bran 5-tier portrait eval with 20 Honkai players, USPTO trademark check on "Catalyst") were still open at the freeze. This is the variant that came closest to being a real game and the variant most in danger of becoming over-specified before it has been play-tested at the moats that matter.

6 — WeaponForge_TFTransistor (current canon)

The bet. Take the pivot addendum's three pillars: (a) Universal Function Sockets — a Transistor-inspired matrix where every drafted Function is multi-purpose, behaving differently as Active / Modifier / Passive (no more anatomical part-type RNG bottleneck); (b) spatial 3-lane auto-runner combat — geometry restores tactical decisions like range, targeting, AoE shape; (c) Magicka cross-hero elemental reactions — the Mage paints a zone Wet+Chilled, the Warrior steps into it with a SLAM, the game detonates a Shatter Reaction for 300% splash. Three pillars × TFT shop drafting × 3-hero squad × ~4-minute stage.

What shipped. A working vertical slice as of 14 Jun (real chibi assets, 329/329 tests). The Phase-5 content batch on 15 Jun added 15-reaction Magicka matrix + 12-Function catalog + tier scaling T1–T5 + hero Ults firing real effects + wave telegraph (Q1–Q5). An adversarial-review workflow (6 readers + per-finding verifiers, 27 agents) then caught 5 spec-fidelity gaps the green sweep had hidden — tests had asserted data shape, not combat effect. Q6 closed those with effect-level tests; final sweep 799/799 across 17 suites. Real-asset-pass branch is canon, build playable end-to-end (Home → F0 forge → 5 stages of 3 auto-battle waves with per-stage forge break → VICTORY or DEFEAT).

What's open right now. Since 16 Jun the work has paused on a design redesign brainstorm: a from-the-research rethink of the core craft. The shipped build is the baseline, not the target. Craft model was resolved 17 Jun (3-layer FORGE → LINEUP → COMBAT; 5 locks including reaction cross-hero, adjacency = Modifier-warp, per-hero frame shapes, role-by-emitter-distance, part vocab = Functions + Modifiers). Open: power-gating — what meters output and times the cascade (Expedition-33 mana/AP-style, casual-blend with hyper-casual research). After that: board specifics + LINEUP detail + legibility → part vocab → Items / Synergies / Catalysts → Wittle-meta lift → balance → spec → plans → TDD build.

What worked — patterns to keep

Design patterns

  • The reveal climax as a payoff invariant. Every variant that had a "you forged something" moment with the 4-second anvil-RING + name banner + arc-to-hero animation produced visible dopamine in playtesters. Variant 4's post-mortem promotes this from a feature to a pattern: every future variant should preserve a payoff moment of similar shape, even if the build-up changes completely.
  • Depth-per-weapon > breadth-of-weapons. Persistent weapons that grow across waves felt good; rotating a new weapon every wave did not. The mechanism can change (re-laps in 4, merge-bumps in 6) but the principle holds.
  • Boss-affinity counter-build telegraph. Every variant uses this and every variant survives it. Pre-stage briefing telegraphs weak: ☀ light · resist: 🪨 earth; the loadout phase is where the player solves the puzzle. This is mechanically cheap and tactically rich.
  • Locked roster + dual-track hero progression. Variant 5's Slot Level 1→200 (inherits) + Hero Mastery 1→100 (per-hero, narrative) is a clean way to separate "the hero gets stronger" from "I get to know this hero." It is also the thing that gives "story-locked roster" — the moat candidate — a mechanical home.
  • The pivot addendum format. When variant 2b pivoted to variant 6, the prototyper wrote a single-doc rationale that explained the three pillars and why the prior architecture was structurally wrong (not just unpolished). That document is what made the pivot reviewable and reversible. Compare with variant 4, whose abandonment was reviewable only because of the post-mortem.

Engineering patterns

  • Self-quitting Test*.tscn headless harnesses with exit code = failure count. Makes "did the whole thing go green?" a single command. Variant 5 has ~640 across 14 suites; variant 6 reached 799 after Q6.
  • RED → GREEN per checkpoint, commit per green. The autonomous overnight runs in 2b and 6 only worked because each task had a clean test boundary. Drift from this rule (variant 4's polish pass) corresponds tightly with drift from playable.
  • Save-schema versioning with explicit migrations + a TestAccountState case per bump. Variant 5 is at v6; variant 6 is at v3. None of those bumps has been silent.
  • Numbers Policy. Numbers in code are starting values, tuned via playtest data. const + ## Why N comment when a value deviates from spec. Prevents the "magic-number drift" failure where someone bumps a constant six weeks later because "it felt better" and now nothing reproduces.
  • AUTOSHOT screenshot helper. The WC_AUTOSHOT env-var pattern lets a windowed Godot scene capture itself to PNG and quit. Made visual QC reproducible — every Mockup PNG has a deterministic "this is what we actually shipped" twin.
  • Adversarial-review workflow. The single best engineering decision in the whole project was running a 27-agent adversarial review against the Phase-5 batch on 15 Jun. It surfaced exactly the gaps a green-sweep was hiding. Make this a default for every non-trivial content batch, not a one-off.

Process patterns

  • NIGHT-LOG / MORNING_REPORT pattern. Autonomous runs write a structured decision log as they go and a stand-alone morning report when they finish. Result: a new chat can resume with full causal context. This is the cheapest single piece of session-continuity insurance in the repo.
  • HANDOFF.md per branch. Every variant that survived a session boundary has one. The variant 6 HANDOFF.md is the canonical example — it has a "Read these to resume, in order" section, a resume sequence, and a paused-vs-active flag.
  • SSOT + ROADMAP + ARCHIVE separation. Each variant's CLAUDE.md explicitly names which files are SSOT, which are forward-work-ok, and which are _archive/-only. Variant 6 even segments _archive/docs-pre-pivot-2026-06-12/ with a HISTORICAL banner on every file.
  • Image-gen cost policy (nano-banana default; nano-banana-pro only on explicit per-call ask). Variant 6's overnight asset pass spent ~$0.46 of the $5 budget on 12 images. The policy is in every project CLAUDE.md and one global; the global enforces it cross-project.
  • Branch naming policy. <sub-project-slug>/<descriptive-keywords>. The variant 5 branches (forgeloop/teammate-deck, forgeloop/catalyst-element-pairs) and the variant 6 branches (weaponforge-tftransistor/post-slice-phase5) are immediately readable six months later. Random-slug branches die in the discoverability bucket.

What failed — antipatterns to avoid

Design antipatterns

  • Polish as substitute for diagnosis. Variant 4 added five layers of Anvil feedback and the playtester signal didn't move. Variant 2b shipped a rich result screen and pivoted the same day. If the loop is wrong, polishing the loop's frame makes the wrong loop sharper. The diagnostic question is "is the core gesture interesting?", not "does the per-tap feedback feel good?"
  • Mode-switched mini-experiences inside a 4-minute session. Modal minigames per craft. Don't. Hearthstone has zero minigames inside Hearthstone. If you do use a minigame, make it the craft, not part of it.
  • Anatomical part-type slots when the combat layer doesn't carry the agency load. The original Head + Hilt + Rune design forced shop RNG to control player progress. Variant 6's Universal Function Sockets fix this by making every rolled Function multi-purpose — there is no "missing hilt" trap.
  • Side-view single-lane combat as the only spatial vocabulary. Range, targeting, AoE shape can't matter on a single lane. Variant 6 moved to 3-lane (and the brainstorm is now considering richer board shapes) precisely because spatial decisions are where item Functions stop being numbers and start being plays.
  • Designing the meta before the moat is felt. Variant 5 has the cleanest meta layer (Catalyst codex, Ember economy, Stage-3 paladin cinematic, prestige skins, battle-pass roadmap) and the least confidence on its core feel. Meta wraps a working core; it cannot substitute for one. Always feel-gate the core before building meta.
  • Treating "the playtester loves the reveal" as "the playtester loves the game." The reveal climax always lands. It is a local maximum of dopamine; it is not the same signal as session-to-session retention. Don't conflate.

Engineering antipatterns

  • The data-vs-effect test gap. The single most expensive engineering lesson in the project. Tests asserted that .tres files had the right fields with the right values; combat never read those fields. Sweep was green; the game was wrong. Always assert the combat effect, not just the data shape. The Q6 fix path is the model: when a Function has an active_max_hits field, the test must spawn enemies, run the tick, and assert hits landed.
  • Parse errors masking the entire suite. A renamed function leaves a stale _ready call → Godot exits 255 with zero test output → looks like a pass. grep -E 'SCRIPT ERROR|Parse Error' on the output before trusting the sweep number.
  • PowerShell *> UTF-16 stdout vs Godot UTF-8 logs. Will silently garble your grep targets. Use Bash > file 2>&1 and Read the file.
  • Godot processes going zombie under contention. Multiple headless runs in parallel from the same project will deadlock on the .godot/imported cache. Stop-Process -Name Godot_v4.6.2-stable_mono_win64_console -Force between runs.
  • Committing .import churn. Godot autosaves .import files on every open and the diff is noise. git add -A sweeps it up by accident. Stage individual files; never -A.
  • class_name resolution races at parse time. Godot 4 cannot reliably resolve class_name MyResource in headless runs unless something has preloaded the script. Workaround: const _MyScript: Script = preload("res://path.gd") and compare res.get_script() == _MyScript.
  • const arrays/dicts are read-only. Tests that hand combat heroes from a const array crash on the first write. .duplicate(true).
  • New PNGs need an --import pass. Drop a .png in assets/ and then load() it in code without importing first → silent null → icons fall back to dots.
  • Writing user://account.json from headless test code. Pollutes the real save. Use in-memory state or temp paths.

Process antipatterns

  • Random branch names. claude/vigorous-chaum-fc300f — no project hint, no scope hint, dies in git branch the moment you have three. The global branch-naming policy now blocks this.
  • Skipping the brainstorm step on "trivial" features. Several mid-project additions to variant 5 went straight to plan without a brainstorm and produced specs that had to be reworked. The 4-step discipline (brainstorm → spec → plan → TDD execute) costs ~30 minutes up front and saves ~3 hours in spec-rework.
  • Holding active design debate in chat instead of docs/superpowers/specs/. Inevitably gets lost on a context boundary. Variant 6's "design redesign brainstorm" survived its pause precisely because it was written down at every step (see the chronologically-named specs 2026-06-15-prototype-direction-design.md, 2026-06-16-craft-mechanic-options-ranked.md, 2026-06-17-craft-model-resolved-and-gating.md).
  • Letting _COMPARISON.md rot. The repo-root variant index hasn't been updated since variant 1. A reader landing in the repo cold has no map. Either keep the index alive or replace it with a "see N_*/CLAUDE.md" pointer.

Game-design fundamentals — adhered vs missed

Adhered to (mostly)

  • Identify the audience first. The Wittle Defender ∩ AFK Journey ∩ Hero Wars cohort was named in variant 1 and never wobbled. Every variant could explain who it was for.
  • Pick one moat candidate per variant. Variant 5: equipment-gacha × locked anime roster. Variant 6: Universal Function Sockets × Magicka cross-hero reactions. The moat is the testable claim.
  • Casual-mobile constraints respected. ~3–4 minute stages. Single-tap ultimate. Auto-combat by default. No keyboard. No drag-everywhere; mobile-friendly tap-to-equip with optional drag.
  • Payoff moments at predictable rhythm. Boss every 5 waves; reveal climax on every craft; stage clear → meta beat.
  • Counter-build telegraph as the cognitive hook. Pre-stage briefing always shows boss weak/resist tags. The loadout phase is the puzzle.

Missed or under-played

  • Feel-gate the core loop before building meta. Variant 5 has a battle-pass roadmap before its core-loop CPI is known. The exit gates are written (D1 ≥ 35%, FM-8 ≥ 6/10, ad CPI ≤ $2.80 — 80% of Wittle benchmark) but not measured.
  • Demand validation cross-cuts. The Marketing-Reels pipeline is designed but no ad-test has run. No CPI signal feeds back into design before the build is far down the cost curve.
  • Player-skill ladder design. Several variants ship with a flat skill ceiling — once you know the combo, the minigame / merge / draft becomes a tax (variant 4's Anvil was the worst offender; variant 6's merge is at risk).
  • The first-15-second rule. Mobile players decide in 10–15s. Variant 4 spent the first 3s of those on a modal minigame for the first craft. Variant 6 currently spends them on a forge break. Neither variant has measured what happens after the first 15s of a cold-start session with a real player.
  • Designing for "I want to come back tomorrow." Stage layout, ult drama, reveal climaxes all hit "fun right now." Variant 5's locked-roster bond + Hero Mastery + scripted cinematics are the only sustained attempt at "I want to come back to this character." That's still unproven.
  • Failure-state design. Variant 6 has permadeath VICTORY/DEFEAT. Variant 5 has boss-retry-with-reforge. These are different bets and we haven't compared them at playtest.

Process discipline — wins, drift, repair

The pattern that actually held

  1. Brainstorm (superpowers:brainstorming) — clarify intent + 2–3 approaches + design + owner approval.
  2. Spec → docs/superpowers/specs/YYYY-MM-DD-<topic>-design.md; commit; owner reviews the file.
  3. Plan (superpowers:writing-plans) → impl plan with ordered tickets; owner approval.
  4. Execute (superpowers:test-driven-development + superpowers:executing-plans or subagent-driven-development) → RED → GREEN → commit per task.

Skip steps only on (a) trivial single-file mechanical work or (b) explicit owner waiver. When the project drifted from this discipline, the next session paid the spec-rework tax. When it held, the project's velocity was the velocity of shipping, not of changing minds.

What broke under load

  • SSOT drift between specs and the build. Repeatedly. Resolved each time with a "the build wins; update the doc" rule baked into each CLAUDE.md.
  • Active vs frozen folder ambiguity. Multiple variants are alive on disk at the same time and a fresh session can't tell which is current. Resolved by aggressive freeze banners + explicit "do NOT extend in this branch" lists in CLAUDE.md.
  • Test-pass false confidence. The data-vs-effect gap. Resolved by Q6 effect-level tests and by adopting an adversarial-review pass for batch landings.
  • Image-gen cost creep. Resolved by a global cost-policy memory and a per-project repeat in CLAUDE.md.

For the next prototyper on this team

If you're inheriting this work — whether to continue variant 6, revive variant 5, or fork a variant 7 — here is the load-bearing condensation.

Pin to your wall
  1. Polish is not a fix. If you're polishing and the playtester signal hasn't moved, stop. Diagnose upstream.
  2. Test the effect, not the data. If your .tres has a field, write a test that proves combat reads it and acts on it.
  3. Preserve the reveal climax. Whatever else changes, keep a payoff moment of the 4-second anvil-RING shape.
  4. Depth-per-weapon > breadth-of-weapons. Persistent + growing beats fresh + disposable.
  5. Spatial agency is non-negotiable. If combat has no positions, the craft has to be insanely deep; if combat has positions, the craft can be cleaner. Pick a lane.
  6. No modal minigames inside the session. Use minigames sparingly, ideally as the whole craft, never as one of N.
  7. Feel-gate the core loop before building meta. Don't write the battle-pass spec until the core CPI is in the ad-test target.
  8. RED → GREEN → commit, per task. Discipline drift = velocity drift, in that order.
  9. Run an adversarial review on every content batch. Green sweeps lie; 27 agents arguing about it don't.
  10. Write the NIGHT-LOG / HANDOFF / pivot-addendum. Future-you and Future-Claude will not remember what current-you decided unless you write it down where the next session can find it.
