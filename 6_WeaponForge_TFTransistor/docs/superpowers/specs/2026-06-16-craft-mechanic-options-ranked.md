## The question this doc answers
The **craft phase is the game's main engagement** (locked 2026-06-16 — see direction doc). It needs a **casual, spatial/combinatorial "arrange things in different ways → emergent power" toy** that:
- **authors Magicka reactions via arrangement** (Water next to Fire = Steam, wired in the craft);
- fuses **Transistor** (one part plays 3 roles: Active/Modifier/Passive) + **TFT** (draft/merge from a slow-roll shop);
- keeps **collectible heroes** meaningful (gacha pull — Ashe the archer);
- stays **casual-mobile + legible on portrait**; combat is **auto-resolved**, reactions are **visual payoff**.

Rejected along the way: literal **gears on a power-core** (user: "don't want it gear-like"); a flat **3-socket menu** (not a toy); whimsical non-forge metaphors (user wants forging/creation/mechanism flavor). Positioning: **closer to Gear Defenders than TFT** (casual collection, not competitive optimization).

## ✅ Tentative winner
**#4 Skeleton attach-points (hero = the frame).** Each hero is a silhouette with a few **attach-nubs**; you clip element-parts onto nubs; **neighbouring nubs react**; each gacha hero has a **different nub layout**, so new heroes = new craft puzzles (collection feeds craft depth). Best fusion of *assembly + heroes + adjacency-reactions*, novel-but-graspable, on-theme. **To be validated** in the next session.

## Full option set, ranked by viability × fit
Fit = casual + forge/mechanism flavor + hero-central + authors-reactions-by-arrangement + not-a-menu. (~30 ideas explored across 3 brainstorm batches.)

### Tier 1 — chosen + nearest (assembly, hero-central, on-theme)
| # | Mechanic | Why it ranks here |
|---|---|---|
| **1** | **Skeleton attach-points (hero=frame)** ⭐ CHOSEN | Heroes are the boards; clip parts on nubs; adjacency reacts; gacha heroes = new layouts. Cleanest assembly + hero + reaction fusion. |
| 2 | **Armor-plates on hero body-grid** | Like #1 but a denser grid (more placement freedom); hero literally wears the build; gacha heroes ship unique body-shapes. |
| 3 | **Segment-snap weapon (linear build)** | Most literal to *WeaponForge*; build the weapon end-to-end, shape = stats, adjacent segments react. Hero identity from who wields it. |

### Tier 2 — strong assembly / forge alternates
| Mechanic | Note |
|---|---|
| Brick baseplate (2D stud interlock) | GD-simple spatial; interlocked neighbours react; shared board → weaker per-hero identity. |
| Tetris-fit loadout (Backpack Battles) | Proven packing puzzle; touching edges react; shared board. |
| Smelting / alloy crucible | Forge-native mixing (Water-ingot + Fire-ingot = Steam-alloy); less of a *spatial* puzzle. |
| Damascus fold (layered billet) | Forge-native; stack + hammer-fold layers; novel + tactile but reaction-author is abstract. |
| Golem / mech-limb assembly | Strong if heroes ARE the assembled golems; risks clashing with the existing hero roster. |
| Bellows & coal-bed | Forge + chain-reaction + adjacency in one; cool, but a distinct 4th UI metaphor. |

## Open sub-questions to resume on #4 (define the toy)
1. **Board:** per-hero silhouette — how many nubs? free-form points vs a small grid of points? how legible on portrait?
2. **Parts:** what is a "part" (an element module)? sizes/shapes? drafted from the slow-roll shop + merged (TFT)?
3. **Adjacency rule:** which nubs count as "neighbours" — connected by the frame's **bones**? does a bone *carry* the reaction along it (chain), or is it pairwise?
4. **Transistor 3-roles:** how do Active/Modifier/Passive map onto attach-points? (e.g. role depends on WHERE clipped — core nub = Active, limb nub = Modifier, trailing nub = Passive?)
5. **Heroes + gacha:** each hero = a unique frame shape (collection = new puzzles). Ashe (archer) = a projectile-flavoured frame.
6. **Combat mapping:** how the assembled hero-frame drives auto-combat + the reaction cascade (still 3 lanes? single field?).
7. **Legibility (the #1 casual risk):** how the player SEES which adjacency wired which reaction (preview lines, highlights, a "this build does X" readout).
